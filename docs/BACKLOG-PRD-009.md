# Backlog: PRD-009 WhatsApp & Signal Stabilization

## Epic: EP-01 WhatsApp & Signal Stabilization & Bug Fixes
**Description:** Resolve critical bugs in the Unified Messaging application regarding missing contacts, broken contact merging, message duplication/loading failures, and missing group chats for both WhatsApp and Signal integrations.

---

## Phase 1: Backend & Data Persistence (Foundation)
*Dependencies: None. Must be completed before Frontend state changes to ensure a reliable data source.*

### User Story: US-01 Reliable Signal Message History
**Description:** As a user, I need to see my historical Signal messages when I open a chat, so that I don't lose context of past conversations.
* **Task: T-01.1 Implement Signal History Fallback**
  * **Location:** `server/signal-server.js`
  * **Details:** Update the `getMessages` handler. If deep history is requested and not supported by `signal-cli`, read from the persisted `data-signal/messages` sqlite/JSON cache file and return the local historical record.

### User Story: US-02 Unified Message ID Schema
**Description:** As a developer, I need unified and deterministic message IDs generated by the server so that the frontend can accurately deduplicate optimistic UI updates.
* **Task: T-02.1 Standardize WhatsApp Message IDs**
  * **Location:** `server/whatsapp-server.js`
  * **Details:** Ensure the fallback logic and standard handlers provide a consistent ID schema that matches frontend expectations. Address the issue where missing `pushname` defaults to a phone number incorrectly if it affects ID generation.
* **Task: T-02.2 Standardize Signal Message IDs**
  * **Location:** `server/signal-server.js`
  * **Details:** Ensure server-side IDs or timestamps are consistently formatted for exact matching against client optimistic IDs.

---

## Phase 2: Frontend Data Handling & State Management
*Dependencies: Phase 1 (Backend). Must be completed to correctly ingest the stabilized server data.*

### User Story: US-03 Accurate Contact & Group Loading
**Description:** As a user, I want to see all my contacts (even unsaved ones) and group chats from WhatsApp and Signal, so that I can communicate with everyone.
* **Task: T-03.1 Fix Contact Filtering (Unsaved Contacts)**
  * **Location:** `App.tsx`
  * **Details:** Remove or heavily loosen the `isProperName` filter to allow pure phone numbers (e.g., `+1234567890`). If a name is a UUID or missing, implement a fallback to "Unknown Number: [Phone]" instead of discarding the contact.
* **Task: T-03.2 Enable WhatsApp & Signal Group Chats**
  * **Location:** `App.tsx`
  * **Details:** Remove the `if (chat.isGroup) continue;` line in `handleWhatsAppChatsLoaded`. Ensure group metadata is correctly mapped (assigning roles like "WhatsApp Group" / "Signal Group" and handling IDs like `@g.us`). Prevent Signal groups without explicit titles from being dropped by regex filters.

### User Story: US-04 Robust Contact Merging
**Description:** As a user, I want my contacts from different platforms merged intelligently so that I don't see duplicate entries for the same person.
* **Task: T-04.1 Implement Phone-Number Matching for Merges**
  * **Location:** `App.tsx`
  * **Details:** Upgrade cross-platform merging logic to use phone-number matching and ID fallbacks, replacing the strict string matching on `normalizeName(u.name)`.
* **Task: T-04.2 Safely Deduplicate Incoming Batches**
  * **Location:** `App.tsx`
  * **Details:** Refactor `setUsers` state updates to safely deduplicate incoming cached data vs live data. Ensure multiple `onChatsLoaded` triggers do not clone "unmerged" identities.

### User Story: US-05 Flawless Message Deduplication
**Description:** As a user, I want my sent messages to appear exactly once and sync properly with the server without duplication.
* **Task: T-05.1 Fix Signal Deduplication Logic**
  * **Location:** `App.tsx`
  * **Details:** Replace the 5-second boundary fingerprint (`Math.floor(m.timestamp.getTime() / 5000)`) in `handleSignalMessagesLoaded` with a sliding window check (e.g., `Math.abs(existing.timestamp - new.timestamp) < 5000`) or, ideally, rely on the unified message ID from Phase 1.
* **Task: T-05.2 Align Optimistic UI Message IDs**
  * **Location:** `src/components/ChatArea.tsx`
  * **Details:** Ensure the optimistic message ID generated on the client strictly mirrors the backend logic. This guarantees that Zustand deduplication works flawlessly via exact `message.id` matching when the server echo arrives.
